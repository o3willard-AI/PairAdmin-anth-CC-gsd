    **D-Bus connection (establish once, reuse):**
    ```go
    package capture

    import (
        "context"
        "fmt"
        "os/exec"
        "strings"
        "log/slog"

        "github.com/godbus/dbus/v5"
        "pairadmin/services/llm/filter"
    )

    const RoleTerminal = uint32(59)

    // ObjectRef is an AT-SPI2 accessible object reference (bus name + object path).
    type ObjectRef struct {
        Name string
        Path dbus.ObjectPath
    }

    // CacheItem represents one item from Cache.GetItems (GTK4/new signature).
    type CacheItem struct {
        Ref           ObjectRef
        AppRef        ObjectRef
        ParentRef     ObjectRef
        IndexInParent int32
        ChildCount    int32
        Interfaces    []string
        Name          string
        Role          uint32
        Description   string
        StateSet      []uint32
    }

    type ATSPIAdapter struct {
        conn        *dbus.Conn  // accessibility bus connection (persistent)
        execCommand func(ctx context.Context, name string, args ...string) *exec.Cmd
        // For testing: injectable D-Bus dialer and session bus getter
        sessionBusFn  func() (*dbus.Conn, error)
        dialFn        func(addr string, opts ...dbus.ConnOption) (*dbus.Conn, error)
    }

    func NewATSPIAdapter() *ATSPIAdapter {
        return &ATSPIAdapter{
            execCommand:  exec.CommandContext,
            sessionBusFn: dbus.SessionBus,
            dialFn:       dbus.Dial,
        }
    }
    ```

    **IsAvailable:** Connect to session bus, call `org.a11y.Bus.GetAddress`. Return true if non-empty address returned. Do NOT check IsEnabled (bus works even when IsEnabled=false, per RESEARCH anti-pattern). Store the address for later use.

    **OnboardingRequired:** Run `gsettings get org.gnome.desktop.interface toolkit-accessibility` via injectable execCommand. Return true if output is not "true". This is an exported method that CaptureManager can query to report adapter status to frontend.

    **connectToA11yBus (internal):** Two-step D-Bus connection per RESEARCH Pattern 3:
    1. Get address via session bus `org.a11y.Bus.GetAddress`
    2. `dbus.Dial(addr)` + `Auth(nil)` + `Hello()`
    3. Store connection in `a.conn`
    4. Only called once from Startup/first Discover; reconnect on error

    **Discover:** Enumerate all names on the accessibility bus via `org.freedesktop.DBus.ListNames`. For each unique name (starting with `:` — skip well-known names per RESEARCH anti-pattern), call `org.a11y.atspi.Cache.GetItems`. Try GTK4 signature `a((so)(so)(so)iiassusau)` first; on type mismatch error, retry with Qt5 signature. Filter CacheItems by `Role == RoleTerminal`. Return PaneInfo list:
    - `ID: fmt.Sprintf("atspi:%s%s", item.Ref.Name, item.Ref.Path)` (per D-03)
    - `AdapterType: "atspi"`
    - `DisplayName: item.Name` (app name or fallback to bus name)

    **Capture:** Given a PaneInfo, parse bus name and object path from the ID. Call `org.a11y.atspi.Text.GetText` with `start=int32(0), end=int32(-1)`. Apply filter pipeline (NewANSIFilter + NewCredentialFilter) with degraded fallback on error.

    **Close:** Close `a.conn` if not nil.

    **Test strategy for atspi_test.go:**
    Create an interface `dbusConnection` that wraps the dbus.Conn methods used (Object, Close). Inject a mock in tests. The mock returns scripted responses for GetAddress, ListNames, Cache.GetItems, and Text.GetText calls. This avoids needing a live D-Bus session for unit tests.

    Alternative simpler approach: make `ATSPIAdapter` accept injectable function fields (like tmux adapter's execCommand) for:
    - `getA11yAddress func() (string, error)` — returns bus address
    - `listBusNames func() ([]string, error)` — returns names on bus
    - `getCacheItems func(busName string) ([]CacheItem, error)` — returns items for a name
    - `getText func(busName string, path dbus.ObjectPath) (string, error)` — returns terminal text

    This approach makes tests straightforward without a full dbus mock framework.
