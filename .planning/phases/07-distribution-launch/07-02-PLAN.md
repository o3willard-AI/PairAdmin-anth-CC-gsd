**Step 1: Install build tools (if missing)**

Install nFPM:
```bash
go install github.com/goreleaser/nfpm/v2/cmd/nfpm@latest
```

Download appimagetool:
```bash
wget -q "https://github.com/AppImage/appimagetool/releases/download/continuous/appimagetool-x86_64.AppImage" -O appimagetool
chmod +x appimagetool
```

**Step 2: Build Wails binary**
```bash
wails build -platform linux/amd64 -tags webkit2_41 -clean
```
Verify output: `ls -la build/bin/pairadmin` — must exist and be executable.

**Step 3: Package .deb and .rpm via nFPM (per D-02 naming)**
```bash
VERSION=1.0.0 nfpm pkg --packager deb --target .
VERSION=1.0.0 nfpm pkg --packager rpm --target .
```
Expected outputs (per D-02):
- `pairadmin_1.0.0_linux_amd64.deb`
- `pairadmin_1.0.0_linux_amd64.rpm`

If nFPM uses a different default naming pattern, rename to match D-02 exactly:
```bash
mv pairadmin_1.0.0_amd64.deb pairadmin_1.0.0_linux_amd64.deb 2>/dev/null || true
mv pairadmin-1.0.0.x86_64.rpm pairadmin_1.0.0_linux_amd64.rpm 2>/dev/null || true
```

**Step 4: Verify .deb contents**
```bash
dpkg --info pairadmin_1.0.0_linux_amd64.deb | grep -E "Depends|Package|Version"
dpkg -c pairadmin_1.0.0_linux_amd64.deb | grep -E "pairadmin|desktop|png"
```
Must show: Package: pairadmin, Version: 1.0.0, Depends includes libwebkit2gtk-4.1-0.

**Step 5: Build AppImage (best-effort per D-09, D-10)**

Create AppDir structure:
```bash
mkdir -p pairadmin.AppDir/usr/bin
cp build/bin/pairadmin pairadmin.AppDir/usr/bin/pairadmin
cp build/linux/pairadmin.desktop pairadmin.AppDir/
cp build/linux/pairadmin.png pairadmin.AppDir/
```

Create AppRun script at `pairadmin.AppDir/AppRun`:
```bash
#!/bin/bash
SELF=$(readlink -f "$0")
HERE="${SELF%/*}"
export PATH="${HERE}/usr/bin:${PATH}"
exec "${HERE}/usr/bin/pairadmin" "$@"
```
Make it executable: `chmod +x pairadmin.AppDir/AppRun`

Build AppImage:
```bash
ARCH=x86_64 ./appimagetool pairadmin.AppDir/ pairadmin_1.0.0_linux_amd64.AppImage || echo "WARNING: AppImage build failed — see Issue #4313"
```

If appimagetool itself fails to run (FUSE issues in container/sandbox), try with `--appimage-extract-and-run`:
```bash
ARCH=x86_64 ./appimagetool --appimage-extract-and-run pairadmin.AppDir/ pairadmin_1.0.0_linux_amd64.AppImage || echo "WARNING: AppImage build failed"
```

**Step 6: Generate SHA256SUMS (per D-03)**
```bash
sha256sum pairadmin_1.0.0_linux_amd64.deb pairadmin_1.0.0_linux_amd64.rpm pairadmin_1.0.0_linux_amd64.AppImage 2>/dev/null > SHA256SUMS || sha256sum pairadmin_1.0.0_linux_amd64.deb pairadmin_1.0.0_linux_amd64.rpm > SHA256SUMS
```

**Step 7: Clean up AppDir (not committed)**
```bash
rm -rf pairadmin.AppDir appimagetool
```

2. **Overview**: 2-3 sentences explaining what PairAdmin does. Core value: "The AI sees exactly what you see in the terminal — automatically, without copy/paste — so assistance is always in context." Mention: works with tmux, GNOME Terminal (via AT-SPI2), supports OpenAI, Anthropic, Ollama, LM Studio, OpenRouter.

3. **## Installation**: Three subsections:

   **### Debian/Ubuntu (.deb) — Recommended**
   ```bash
   # Install runtime dependencies
   sudo apt install -y libwebkit2gtk-4.1-0 at-spi2-core

   # Install PairAdmin
   sudo dpkg -i pairadmin_1.0.0_linux_amd64.deb

   # Launch
   pairadmin
   ```

   **### Fedora/RHEL (.rpm)**
   ```bash
   # Install runtime dependencies
   sudo dnf install -y webkit2gtk4.1 at-spi2-atk

   # Install PairAdmin
   sudo rpm -i pairadmin_1.0.0_linux_amd64.rpm

   # Launch
   pairadmin
   ```

   **### AppImage**
   ```bash
   chmod +x pairadmin_1.0.0_linux_amd64.AppImage
   ./pairadmin_1.0.0_linux_amd64.AppImage
   ```
   Include a note: "> **Note:** The AppImage may fail to launch due to WebKit2GTK subprocess path limitations ([Wails Issue #4313](https://github.com/wailsapp/wails/issues/4313)). If you encounter errors, use the `.deb` or `.rpm` package instead."

4. **## Verifying Downloads**: Per D-03:
   ```bash
   sha256sum --check SHA256SUMS
   ```
   Note (per D-04): "GPG signing is planned for future releases."

5. **## Prerequisites**: Before using PairAdmin, you need:
   - A terminal multiplexer: `tmux` (recommended) or a GTK3-based terminal (GNOME Terminal)
   - An LLM provider: Ollama (local, private — no data leaves your machine), or a cloud API key for OpenAI, Anthropic, OpenRouter, or LM Studio

6. **## Quick Start**:
   1. Launch PairAdmin: `pairadmin`
   2. Open Settings (gear icon in status bar) and configure your LLM provider
   3. Start a tmux session in another terminal: `tmux new -s work`
   4. PairAdmin auto-discovers the tmux pane — you'll see it in the left sidebar
   5. Type a question in the chat input — the AI has full context of your terminal

7. **## Building from Source**:
   ```bash
   # Install build dependencies
   sudo ./scripts/install-deps.sh

   # Install Go 1.24+ and Node.js 20+
   # Install Wails CLI: go install github.com/wailsapp/wails/v2/cmd/wails@latest

   # Build
   wails build -platform linux/amd64 -tags webkit2_41
   # Binary at: build/bin/pairadmin
   ```
